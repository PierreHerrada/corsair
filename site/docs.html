Overview

Corsair is an open-source AI software engineering platform. It connects Slack, Jira, GitHub, and Claude Code into a unified autonomous workflow. Tag @Corsair in Slack, and the platform plans the work, writes the code, and opens a pull request — with human approval at every stage.

The entire platform ships as a single Docker image using supervisord to run an nginx reverse proxy (serving the React frontend) and a uvicorn server (running the FastAPI backend with WebSocket and Slack bot support).

Prerequisites

  • Docker and Docker Compose (for containerised setup)
  • Python 3.12+ (for local backend development)
  • Node.js 20+ and npm (for local frontend development)
  • PostgreSQL 15+ (or use Docker Compose which provides it)
  • An Anthropic API key for Claude

Quick Start

The fastest way to get Corsair running locally is with Docker Compose:

terminal 
# Clone the repository
git clone https://github.com/PierreHerrada/clipper-ai.git
cd clipper-ai

# Copy and configure environment variables
cp .env.example .env
# Edit .env with your credentials (see Environment Variables below)

# Start the full stack
docker-compose up --build

Once started, the web UI is available at http://localhost and the API at http://localhost:8000.

Integrations are opt-in. If Slack, Jira, or GitHub env vars are missing, those integrations stay inactive — the app still boots and works without them.

Slack Setup

  1. Create a Slack app at api.slack.com
  2. Enable Socket Mode and generate an app-level token (xapp-...)
  3. Under OAuth & Permissions, add the scopes: app_mentions:read, chat:write
  4. Install the app to your workspace and copy the Bot User OAuth Token (xoxb-...)
  5. Set SLACK_BOT_TOKEN and SLACK_APP_TOKEN in your .env

Jira Setup

  1. Generate an API token at id.atlassian.com
  2. Set JIRA_BASE_URL (e.g. https://yourorg.atlassian.net), JIRA_EMAIL, JIRA_API_TOKEN, and JIRA_PROJECT_KEY

GitHub Setup

  1. Create a personal access token with the repo scope
  2. Set GITHUB_TOKEN and GITHUB_ORG in your .env

Local Development

For development, you can run the backend and frontend separately. Make sure PostgreSQL is running locally or via Docker.

Backend

backend 
cd backend

# Install dependencies
pip install -r requirements.txt

# Run the dev server
uvicorn app.main:app --reload --port 8000

# Lint & format
ruff check app/ tests/
ruff format app/ tests/

Frontend

frontend 
cd frontend

# Install dependencies
npm ci

# Start dev server (hot-reload)
npm run dev

# Lint & format
npm run lint
npm run format

# Production build
npm run build

Testing

Both backend and frontend enforce a minimum of 80% code coverage.

tests 
# Backend tests with coverage
cd backend
pytest tests/ --cov=app --cov-fail-under=80

# Frontend tests with coverage
cd frontend
npm test -- --coverage
Never call real external APIs in tests. Always mock Slack, Jira, GitHub, and Anthropic calls. The CI pipeline enforces this.

Architecture Overview

Corsair runs as a single Docker image managed by supervisord, which starts two processes:

  • nginx on port 80 — serves the React static build and proxies API/WebSocket requests to uvicorn
  • uvicorn on port 8000 — runs the FastAPI application (REST API, WebSocket server, Slack bot)

The backend connects to a PostgreSQL database (local for dev, RDS for production) and communicates with external services through a plugin-based integration registry.

Three-Stage Agent Pipeline

Every task goes through three stages, each manually triggered from the web UI:

  1. Plan — Claude Code analyses the task, reads the codebase, and produces a structured implementation plan
  2. Work — Claude Code executes the plan: writes code, creates branches, and opens a pull request on GitHub
  3. Review — Claude Code reviews the diff, checks for issues, and validates the implementation against the original task

Each stage streams logs in real-time via WebSocket. Token usage and costs are recorded per stage.

Integrations

Integrations live in backend/app/integrations/, each in its own directory. Built-in integrations:

  • Slack — Socket Mode bot that listens for @Corsair mentions and creates tasks
  • Jira — Creates and updates Jira tickets for each task
  • GitHub — Opens pull requests with branch management and Jira linking

To add a new integration, create a new directory under backend/app/integrations/ and follow the patterns in INTEGRATIONS.md.

Docker

The multi-stage Dockerfile builds both the React frontend and the Python backend into a single image:

docker 
# Build and run with Docker Compose
docker-compose up --build

# Run tests inside the container
docker-compose run backend pytest

# Build the image standalone
docker build -t corsair .

Production Deployment

Corsair is designed to run on AWS ECS Fargate. The infra/ directory contains:

  • ecs-task-definition.json — ECS task definition for Fargate
  • nginx.conf — Nginx config that proxies /api and /ws to uvicorn
  • supervisord.conf — Process manager configuration
  • deploy.sh — Deployment script

Set ENVIRONMENT=production and configure your DATABASE_URL to point to an RDS PostgreSQL instance. All secrets should be managed through environment variables — never hardcoded.

CI/CD is handled by GitHub Actions. The pipeline runs backend and frontend tests, enforces 80% coverage, and uploads coverage reports to Codecov on every push and pull request.
